# M6 — 백업·복구와 `.nfab` (요약)

**상위:** [M6_PDM_LITE.md](./M6_PDM_LITE.md).

## 단일 진실 소스

- **워크스페이스 편집 본문:** `.nfab` 직렬화 트리(히스토리·씬·어셈블리·`meta.nexyfabPdm` 포함).  
- **클라우드:** 동일 페이로드를 `nf_projects.scene_data`에 저장. 서버 스냅샷 `nf_project_versions`는 최대 20개 롤링.

## 백업

| 경로 | 설명 |
|------|------|
| **로컬 파일** | 사용자가 `.nfab`로 “다른 이름 저장” — 전체 상태 포함, 오프라인 복구에 적합. |
| **클라우드** | 로그인 + 자동/수동 PATCH — 계정·서버 보존 정책에 따름. |
| **버전 패널** | 브라우저 로컬 히스토리(별도 용량 한도) — 단기 복구. |

## 복구 시나리오

1. **파일만 있음:** Shape Generator에서 `.nfab` 열기 → `meta`·PDM 스트립 복원.  
2. **클라우드만 있음:** 대시보드 또는 `?projectId=` 로 열기 → 서버 `scene_data` 적용. **뷰어**는 `canEdit: false`로 저장 불가.  
3. **409 충돌:** 토스트의 “서버에서 다시 불러오기”로 URL 재진입.

## 마이그레이션

- `.nfab`에 알 수 없는 `meta` 키는 **보존**한다(`buildSerializedMeta` 병합).  
- `nexyfabPdm` 스키마 확장 시: 기본값으로 이전 파일을 읽을 수 있게 파서를 확장한다.
